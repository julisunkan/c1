What is Vulnerability Assessment?

Vulnerability assessment is the systematic process of identifying, analyzing, and reporting security weaknesses in systems, applications, and networks. It forms a critical component of the overall security testing methodology.

Key Difference: Vulnerability assessment identifies potential security weaknesses, while penetration testing attempts to exploit those weaknesses to demonstrate real-world impact.

Types of Vulnerability Assessments

1. Network Vulnerability Assessment

  • Network infrastructure scanning
  • Service and protocol analysis
  • Configuration assessment
  • Patch level verification

2. Web Application Assessment

  • Session management issues

3. Database Assessment

  • Database configuration review
  • Access control analysis

4. Wireless Assessment

  • Encryption strength analysis

Vulnerability Scanning Process

Phase 1: Pre-Scan Preparation

Planning and Scoping

  • Define scope: Identify target systems and networks
  • Timing coordination: Schedule scans during approved windows

  • Baseline establishment: Document current system state

Tool Selection

  • Authenticated vs. Unauthenticated: Choose scanning approach
  • Tool capabilities: Match tools to assessment requirements
  • Performance considerations: Balance thoroughness with system impact

Phase 2: Discovery and Enumeration

# Network discovery with Nmap nmap -sn 192.168.1.0/24 # Ping sweep nmap -sS -O 192.168.1.1-254 # SYN scan with OS detection nmap -sV 192.168.1.10 # Service version detection nmap -A 192.168.1.10 # Aggressive scan # Port scanning techniques nmap -sS target # SYN scan (stealth) nmap -sT target # TCP connect scan nmap -sU target # UDP scan nmap -sF target # FIN scan nmap -sX target # Xmas scan

Automated Vulnerability Scanning Tools

Nessus

Nessus is a comprehensive vulnerability scanner that can identify thousands of vulnerabilities, malware, and configuration issues across various platforms and applications.

Nessus Advantages:

  • Comprehensive vulnerability database
  • Regular plugin updates
  • Detailed reporting capabilities
  • Low false positive rate
  • Integration with other security tools

OpenVAS

Open-source vulnerability scanner with community-driven updates:

# OpenVAS basic commands sudo gvm-setup # Initial setup sudo gvm-start # Start services sudo gvm-stop # Stop services sudo greenbone-nvt-sync # Update vulnerability tests

OpenVAS Features:

  • Free and open-source
  • Web-based management interface
  • Customizable scan configurations
  • Support for credentialed scans
  • Extensive reporting options

Nikto

Web server scanner focused on dangerous files and configurations:

# Nikto scanning examples nikto -h http://target.com # Basic web server scan nikto -h http://target.com -p 80,443 # Scan specific ports nikto -h http://target.com -o report.txt # Save output to file nikto -h target.com -Tuning 1,2,3 # Specific test categories

Nmap Advanced Scanning Techniques

Service and Version Detection

# Service enumeration nmap -sV target # Version detection nmap -sC target # Default script scan nmap --script vuln target # Vulnerability scripts nmap --script="not intrusive" target # Non-intrusive scripts nmap -A target # Aggressive scan (OS, version, scripts) # Custom NSE scripts nmap --script http-enum target # Web directory enumeration nmap --script smb-vuln-* target # SMB vulnerability checks nmap --script ssl-* target # SSL/TLS assessment

Timing and Performance

# Timing templates nmap -T0 target # Paranoid (very slow) nmap -T1 target # Sneaky nmap -T2 target # Polite nmap -T3 target # Normal (default) nmap -T4 target # Aggressive nmap -T5 target # Insane (very fast) # Custom timing nmap --min-rate 1000 target # Minimum packet rate nmap --max-rate 5000 target # Maximum packet rate nmap --max-retries 2 target # Retry attempts

# ZAP command line options zap.sh -cmd # Command line mode zap.sh -daemon # Daemon mode zap.sh -quickurl http://target.com # Quick scan zap.sh -quickout report.html # Output report # ZAP scripting zap-cli quick-scan http://target.com # Quick automated scan zap-cli spider http://target.com # Spider the application zap-cli active-scan http://target.com # Active vulnerability scan

Burp Suite Scanning

Professional web application security testing platform:

  • Proxy: Intercept and modify HTTP/HTTPS traffic

Database Vulnerability Assessment

SQLmap

Automated SQL injection and database takeover tool:

# Basic SQLmap usage sqlmap -u "http://target.com/page?id=1" # Test URL for SQL injection sqlmap -u "target.com" --forms # Test forms automatically sqlmap -u "target.com" --dbs # Enumerate databases sqlmap -u "target.com" -D db --tables # Enumerate tables sqlmap -u "target.com" -D db -T table --columns # Enumerate columns # Advanced techniques sqlmap -u "target.com" --cookie="session=abc" # Use cookies sqlmap -u "target.com" --data="post=data" # POST data sqlmap -u "target.com" --proxy="http://proxy:8080" # Use proxy

Vulnerability Classification and Scoring

Common Vulnerability Scoring System (CVSS)

Industry standard for assessing vulnerability severity:

CVSS v3.1 Metrics

  • Base Score: Intrinsic characteristics (0.0-10.0)
  • Temporal Score: Time-dependent factors
  • Environmental Score: Organization-specific impact

Base Score Components

Metric Values Description
Attack Vector Network, Adjacent, Local, Physical How vulnerability is exploited
Attack Complexity Low, High Conditions required for exploitation
Privileges Required None, Low, High Privilege level needed
User Interaction None, Required User involvement needed
Scope Unchanged, Changed Impact beyond vulnerable component
Impact (C/I/A) None, Low, High Confidentiality, Integrity, Availability

Vulnerability Categories

Network Vulnerabilities

  • Unpatched Systems: Missing security updates
  • Weak Protocols: Telnet, FTP, HTTP (unencrypted)
  • Default Credentials: Unchanged default passwords
  • Open Services: Unnecessary services exposed
  • Misconfiguration: Insecure service settings

Web Application Vulnerabilities

  • Injection Flaws: SQL, NoSQL, OS command injection
  • Broken Authentication: Session management issues
  • Sensitive Data Exposure: Unencrypted data transmission
  • XML External Entities (XXE): XML parsing vulnerabilities
  • Broken Access Control: Authorization bypass
  • Security Misconfigurations: Default or weak configurations
  • Cross-Site Scripting (XSS): Client-side injection
  • Insecure Deserialization: Object injection attacks
  • Known Vulnerable Components: Outdated libraries
  • Insufficient Logging: Inadequate monitoring

False Positives and Validation

Common False Positive Sources

  • Version-based detection: Scanner assumes vulnerability based on version
  • Banner information: Modified or inaccurate service banners
  • Network conditions: Timeouts misinterpreted as vulnerabilities
  • Custom applications: Unique implementations not recognized

Validation Techniques

  1. Manual verification: Manually test reported vulnerabilities
  2. Proof of concept: Develop working exploits
  3. Configuration review: Examine actual system settings
  4. Multiple tools: Cross-validate with different scanners
Important: Always validate scanner results before reporting. False positives can undermine the credibility of your assessment.
