Module 5: Vulnerability Scanning and Assessment

Vulnerability scanning is a systematic approach to identifying security weaknesses in systems and applications. This module covers automated scanning tools, manual assessment techniques, and vulnerability prioritization for effective penetration testing.

Learning Objectives

  • • Master automated vulnerability scanning tools and techniques
  • • Understand different types of vulnerabilities and their classifications
  • • Learn manual vulnerability assessment and validation methods
  • • Implement effective vulnerability management workflows
  • • Prioritize vulnerabilities based on risk and exploitability
  • • Generate comprehensive vulnerability assessment reports

Vulnerability Classifications

Understanding vulnerability types and severity classifications helps prioritize remediation efforts effectively.

CVSS Scoring System

Critical (9.0-10.0): Immediate action required
High (7.0-8.9): High priority remediation
Medium (4.0-6.9): Moderate risk level
Low (0.1-3.9): Lower priority

Common Vulnerability Types

  • • Buffer Overflows: Memory corruption vulnerabilities
  • • SQL Injection: Database query manipulation
  • • XSS: Cross-site scripting attacks
  • • CSRF: Cross-site request forgery
  • • Path Traversal: Directory traversal attacks
  • • Privilege Escalation: Unauthorized access elevation

Nmap Scripting Engine (NSE)

NSE provides powerful vulnerability detection capabilities through custom scripts and community contributions.

Vulnerability Detection Scripts

nmap --script vuln target.com
nmap --script smb-vuln* target.com
nmap --script http-vuln* target.com
nmap --script ssl-* target.com

Specific Vulnerability Checks

ms17-010 EternalBlue SMB
heartbleed OpenSSL vulnerability
shellshock Bash CGI vulnerability
ssl-poodle SSL 3.0 vulnerability

Database Vulnerability Assessment

Database systems require specialized scanning techniques to identify configuration weaknesses and security flaws.

Database Security Checks

Authentication Weaknesses
  • • Default or blank passwords
  • • Weak password policies
  • • Unnecessary user accounts
  • • Excessive privileges
Configuration Issues
  • • Unpatched database software
  • • Insecure network configurations
  • • Missing encryption settings
  • • Inadequate logging and monitoring

Manual Vulnerability Assessment

Manual testing complements automated scanning by identifying logic flaws and complex vulnerabilities.

Manual Testing Techniques

  • • Source Code Review: Static analysis of application code
  • • Configuration Analysis: Security settings review
  • • Business Logic Testing: Workflow and process flaws
  • • Authentication Testing: Login mechanism analysis
  • • Session Management: Token and session security
  • • Input Validation: Data handling verification

Validation Methods

  • • Proof of Concept: Demonstrate exploitability
  • • False Positive Analysis: Verify scanner results
  • • Impact Assessment: Evaluate potential damage
  • • Exploit Development: Create working exploits
  • • Remediation Testing: Verify fixes effectiveness
  • • Regression Testing: Ensure no new issues

Vulnerability Prioritization

Effective prioritization ensures critical vulnerabilities receive immediate attention while managing resource constraints.

Risk Assessment Factors

Technical Factors
  • • CVSS base score
  • • Exploitability rating
  • • Attack complexity
  • • Required privileges
  • • User interaction needed
Business Factors
  • • Asset criticality
  • • Data sensitivity
  • • Regulatory requirements
  • • Business impact
  • • Remediation cost

Scanner Configuration and Tuning

Proper scanner configuration maximizes effectiveness while minimizing false positives and system impact.

Scan Configuration

  • • Scope Definition: Target systems and networks
  • • Timing Settings: Scan frequency and duration
  • • Intensity Levels: Aggressive vs. safe scanning
  • • Authentication: Credentialed vs. uncredentialed
  • • Plugin Selection: Relevant vulnerability checks
  • • Exclusions: Critical systems and maintenance windows

Performance Optimization

  • • Network Bandwidth: Throttling and rate limiting
  • • Concurrent Scans: Parallel scanning optimization
  • • Resource Usage: Memory and CPU considerations
  • • Scan Policies: Customized scanning profiles
  • • Update Management: Plugin and signature updates
  • • Result Storage: Database optimization and archiving

False Positive Management

Effective false positive management reduces noise and focuses attention on genuine security issues.

False Positive Reduction Strategies

  • • Scanner Tuning: Adjust sensitivity and detection rules
  • • Manual Verification: Validate high-priority findings
  • • Baseline Establishment: Create known-good configurations
  • • Exception Management: Document accepted risks
  • • Contextual Analysis: Consider environmental factors
  • • Historical Trending: Track false positive patterns

Key Takeaways

Vulnerability scanning and assessment provide the foundation for effective security programs. This module covered automated scanning tools, manual assessment techniques, and comprehensive vulnerability management processes. Effective vulnerability management requires combining automated tools with manual validation, proper prioritization based on business risk, and systematic tracking of remediation efforts. Remember that vulnerability scanning is an ongoing process, not a one-time activity, and should be integrated into continuous security monitoring programs.
