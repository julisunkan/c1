Learning Objectives

Lab Architecture Design

A well-designed lab architecture mimics real-world networks while maintaining isolation from production systems.

Network Segmentation Strategy

Management Network (192.168.1.0/24)
  • • Hypervisor management interfaces
  • • Administrative access to lab systems
  • • Monitoring and logging servers
  • • Backup and update services
DMZ Network (192.168.10.0/24)
  • • Web servers and public-facing services
  • • Email servers and DNS servers
  • • Load balancers and reverse proxies
  • • Jump boxes and bastion hosts
Internal Network (192.168.20.0/24)
  • • Domain controllers and file servers
  • • Database servers and application servers
  • • Workstations and user endpoints
  • • Network attached storage devices

Network Configuration

Proper network configuration ensures realistic testing scenarios while maintaining security isolation.

Virtual Network Setup

Host-Only Networks

Isolated from physical network, VMs can communicate with each other and host

NAT Networks

VMs share host's IP address, can access internet but isolated from physical network

Bridged Networks

VMs appear as separate devices on physical network (use with caution)

Internal Networks

Completely isolated, no host or internet access

Installing and Configuring Kali Linux

Kali Linux is the de facto standard for penetration testing distributions, packed with security tools.

Installation Steps

  1. 1. Download Kali Linux ISO: Get the latest version from official website
  2. 2. Create Virtual Machine: Allocate at least 4GB RAM and 40GB storage
  3. 3. Boot from ISO: Start installation process
  4. 4. Configure Network: Set up network interfaces and hostname
  5. 5. Create User Account: Avoid using root as primary account
  6. 6. Install VMware Tools: Enable enhanced features and integration
  7. 7. Update System: Run apt update && apt full-upgrade
  8. 8. Install Additional Tools: Add specialized tools as needed

Lab Security and Isolation

Maintaining proper isolation prevents lab activities from affecting production networks.

Security Best Practices

  • • Network Isolation: Use host-only or internal networks for sensitive testing
  • • Firewall Rules: Block lab traffic from reaching production networks
  • • VM Snapshots: Create clean states for repeatable testing
  • • Resource Limits: Prevent lab VMs from consuming all host resources
  • • Malware Handling: Use isolated environments for malware analysis
  • • Data Protection: Encrypt sensitive lab data and configurations
