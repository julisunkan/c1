File Permissions and Ownership

Understanding Permissions

Linux uses a three-tier permission system:

  • Owner (u): File creator/owner
  • Group (g): Members of the file's group
  • Others (o): All other users

Permission Types

  • Read (r/4): View file contents or list directory
  • Write (w/2): Modify file or create/delete files in directory
  • Execute (x/1): Run file as program or access directory

Permission Commands

# View detailed permissions ls -la # Change permissions (symbolic) chmod u+x filename # Add execute for owner chmod g-w filename # Remove write for group chmod o=r filename # Set read-only for others # Change permissions (numeric) chmod 755 filename # rwxr-xr-x chmod 644 filename # rw-r--r-- # Change ownership chown user:group filename chgrp group filename
Security Note: Misconfigured permissions are a common source of privilege escalation vulnerabilities. Always review file permissions carefully.

Security Best Practices

  1. Use sudo instead of root: Minimize time spent as root user
  2. Secure SSH: Use key-based authentication
  3. Regular updates: Keep system and packages updated
  4. Monitor logs: Regularly check system logs
  5. File permissions: Follow principle of least privilege
  6. Backup important data: Regular backups of critical files
  7. Use strong passwords: Complex passwords for all accounts
Remember: The command line is your most powerful tool in penetration testing. Mastering these fundamentals will significantly improve your efficiency and effectiveness.
