Module 4: Reconnaissance and Information Gathering

Reconnaissance is the foundation of successful penetration testing. This module teaches systematic information gathering techniques to identify potential attack vectors and understand target environments before launching attacks.

Learning Objectives

  • • Master passive and active reconnaissance techniques
  • • Perform comprehensive network discovery and enumeration
  • • Analyze target infrastructure and identify attack surfaces
  • • Use OSINT (Open Source Intelligence) for information gathering
  • • Document findings systematically for planning attack strategies
  • • Understand legal and ethical boundaries in reconnaissance

Reconnaissance Methodology

A systematic approach to reconnaissance ensures comprehensive coverage while maintaining stealth and staying within legal boundaries.

Passive Reconnaissance

  • • No direct interaction with target systems
  • • Difficult to detect or attribute
  • • Uses publicly available information
  • • OSINT techniques and search engines
  • • Social media and public records
  • • DNS and WHOIS queries

Active Reconnaissance

  • • Direct interaction with target systems
  • • More detailed information gathering
  • • Higher risk of detection
  • • Port scanning and service enumeration
  • • Banner grabbing and fingerprinting
  • • Vulnerability scanning

Open Source Intelligence (OSINT)

OSINT involves collecting information from publicly available sources to build a comprehensive picture of the target organization.

OSINT Sources and Techniques

Search Engine Intelligence
  • • Google Dorking: site:target.com filetype:pdf
  • • Bing/Yahoo: Alternative search engines for different results
  • • Shodan: Search engine for internet-connected devices
  • • Censys: Internet-wide scanning and search platform
Social Media Intelligence
  • • LinkedIn: Employee information and company structure
  • • Twitter/X: Real-time information and sentiment
  • • Facebook: Personal and professional connections
  • • GitHub: Source code and configuration files

Domain and DNS Reconnaissance

DNS information provides valuable insights into network infrastructure and potential attack vectors.

DNS Enumeration Tools

nslookup target.com
dig target.com ANY
host -a target.com
dnsrecon -d target.com

DNS Record Types

A: IPv4 address
AAAA: IPv6 address
MX: Mail exchange
NS: Name server
TXT: Text records (SPF, DKIM)

Network Discovery and Scanning

Network scanning reveals active hosts, open ports, and running services within the target infrastructure.

Nmap Scanning Techniques

Host Discovery
nmap -sn 192.168.1.0/24

Ping sweep to identify live hosts

TCP SYN Scan
nmap -sS -p 1-65535 target.com

Stealthy port scan using SYN packets

Service Detection
nmap -sV -p 80,443,22,21 target.com

Identify service versions on open ports

OS Detection
nmap -O target.com

Attempt to identify operating system

Service Enumeration

Once services are identified, detailed enumeration reveals configuration details and potential vulnerabilities.

Web Services (80/443)

nikto -h http://target.com
dirb http://target.com
gobuster dir -u http://target.com -w wordlist.txt
whatweb http://target.com

SSH Services (22)

ssh-keyscan target.com
nmap --script ssh-hostkey target.com
nmap --script ssh-auth-methods target.com
hydra -l admin -P passwords.txt ssh://target.com

SMB and NetBIOS Enumeration

Windows environments often expose valuable information through SMB and NetBIOS services.

SMB Enumeration Commands

enum4linux -a target.com Comprehensive SMB enumeration
smbclient -L //target.com -N List available shares
rpcclient -U "" -N target.com Connect via RPC for enumeration
nmap --script smb-enum-shares target.com Nmap SMB share enumeration

Database Service Enumeration

Database services often contain sensitive information and may have weak authentication mechanisms.

MySQL (3306)

nmap --script mysql-info target.com
mysql -h target.com -u root -p
nmap --script mysql-enum target.com

MSSQL (1433)

nmap --script ms-sql-info target.com
sqsh -S target.com -U sa
nmap --script ms-sql-empty-password target.com

Email and Communication Services

Email services provide valuable information about users and may have authentication vulnerabilities.

Email Service Enumeration

SMTP Enumeration (25)
nmap --script smtp-enum-users --script-args smtp-enum-users.methods={VRFY,EXPN,RCPT} target.com
POP3 Testing (110)
telnet target.com 110
IMAP Testing (143)
nmap --script imap-capabilities target.com

Web Application Reconnaissance

Web applications present a large attack surface and require specialized reconnaissance techniques.

Technology Identification

  • • Wappalyzer: Browser extension for tech stack
  • • Whatweb: Command-line web technology scanner
  • • BuiltWith: Online technology profiler
  • • HTTP Headers: Server and framework information
  • • robots.txt: Discover hidden directories
  • • sitemap.xml: Map website structure

Content Discovery

  • • Directory Brute Force: Find hidden paths
  • • File Extension Discovery: Backup and config files
  • • Parameter Discovery: Hidden form parameters
  • • Subdomain Enumeration: Find additional services
  • • Virtual Host Discovery: Identify hosted sites
  • • API Endpoint Discovery: Hidden API functionality

Subdomain Enumeration

Subdomains often host development environments, APIs, and services with weaker security controls.

Subdomain Discovery Tools

sublist3r -d target.com Passive subdomain enumeration
amass enum -d target.com Comprehensive subdomain discovery
gobuster dns -d target.com -w subdomains.txt Brute force subdomain discovery
dnsrecon -d target.com -t brt -D subdomains.txt DNS brute force enumeration

Social Engineering Information Gathering

Understanding the human element is crucial for comprehensive security assessments.

Employee Information

  • • LinkedIn profiles: Job titles and connections
  • • Company websites: Staff directories and bios
  • • Social media: Personal information and habits
  • • Professional networks: Industry connections
  • • Conference presentations: Technical expertise
  • • Email patterns: Naming conventions

Company Intelligence

  • • Technology stack: Job postings reveal tech used
  • • Physical locations: Office addresses and layouts
  • • Business relationships: Partners and vendors
  • • Financial information: Public filings and reports
  • • News and press: Recent developments

Metadata Analysis

Document metadata often contains valuable information about internal systems and users.

Metadata Extraction Tools

exiftool document.pdf Extract metadata from files
metagoofil -d target.com -t pdf,doc,xls Search and analyze documents
foca Windows-based metadata analysis tool

Reconnaissance Documentation

Systematic documentation of reconnaissance findings is crucial for attack planning and reporting.

Information Organization

  • • Network Infrastructure: IP ranges, domains, subdomains
  • • Service Inventory: Open ports, service versions, configurations
  • • Application Catalog: Web apps, APIs, file structures
  • • User Database: Email addresses, usernames, social profiles
  • • Technology Stack: Operating systems, frameworks, databases
  • • Security Controls: Firewalls, IDS/IPS, monitoring systems

Custom Automation

  • • Report generation: Automated documentation

Legal and Ethical Considerations

Understanding legal boundaries ensures reconnaissance activities remain within authorized scope.

Reconnaissance Guidelines

  • • Stay in Scope: Only target systems explicitly authorized
  • • Passive First: Begin with passive techniques to minimize risk
  • • Rate Limiting: Avoid overwhelming target systems
  • • Document Everything: Maintain detailed logs of activities
  • • Respect Privacy: Handle personal information responsibly
  • • Legal Compliance: Follow local laws and regulations

Practical Exercises

Hands-on practice reinforces reconnaissance techniques and builds systematic investigation skills.

Exercise 1: Passive Reconnaissance

Perform comprehensive passive reconnaissance on a target domain:

  • • Gather domain registration information (WHOIS)
  • • Enumerate subdomains using passive techniques
  • • Collect email addresses and social media profiles
  • • Analyze company documents for metadata
Exercise 2: Network Discovery

Map network infrastructure using active scanning:

  • • Perform host discovery on target networks
  • • Conduct comprehensive port scans
  • • Enumerate services and versions
  • • Create network topology diagram
Exercise 3: Web Application Reconnaissance

Thoroughly analyze web application attack surface:

  • • Identify web technologies and frameworks
  • • Discover hidden directories and files
  • • Enumerate parameters and endpoints
  • • Map application functionality and data flow

Key Takeaways

Effective reconnaissance forms the foundation of successful penetration testing. This module covered passive and active information gathering techniques, from OSINT collection to network scanning and service enumeration. Systematic documentation and automation improve efficiency and ensure comprehensive coverage. Remember that reconnaissance should always begin with passive techniques and progress to active scanning only when authorized. The information gathered during this phase directly influences the success of subsequent attack phases.
