Chapter 1: Password Hashing Algorithms

Different hashing algorithms provide varying levels of security and computational requirements

Understanding how passwords are hashed and stored is crucial for both defensive and offensive security operations. Different algorithms provide varying levels of protection against cracking attempts.

Weak Hashing Algorithms

  • ‚Ä¢ MD5: Fast but cryptographically broken
  • ‚Ä¢ SHA-1: Vulnerable to collision attacks
  • ‚Ä¢ Plain SHA-256/512: Too fast for passwords
  • ‚Ä¢ DES: Outdated encryption standard
  • ‚Ä¢ LM Hashes: Extremely weak Windows legacy
  • ‚Ä¢ NTLM: Vulnerable to various attacks

Strong Hashing Algorithms

  • ‚Ä¢ bcrypt: Adaptive hashing with salt
  • ‚Ä¢ scrypt: Memory-hard function
  • ‚Ä¢ Argon2: Modern password hashing winner
  • ‚Ä¢ PBKDF2: Key derivation with iterations
  • ‚Ä¢ Blowfish: Basis for bcrypt algorithm
  • ‚Ä¢ yescrypt: Enhanced scrypt variant

Chapter 2: Password Cracking Techniques

Various attack methods target different aspects of password security

Password cracking involves various techniques ranging from simple dictionary attacks to sophisticated hybrid approaches combining multiple methodologies.

Dictionary Attacks

Use predefined wordlists to attempt common passwords

hashcat -m 0 -a 0 hashes.txt rockyou.txt - MD5 dictionary attack

john --wordlist=passwords.txt hashes.txt - John the Ripper dictionary

hydra -L users.txt -P passwords.txt ssh://target - Online dictionary attack

Brute Force Attacks

Systematically try all possible character combinations

hashcat -m 0 -a 3 hashes.txt ?a?a?a?a?a?a - 6-character brute force

john --incremental hashes.txt - Incremental brute force

crunch 8 8 -t @@@@%%%% | hashcat -m 0 hashes.txt - Pattern-based

Rule-Based Attacks

Apply transformation rules to base passwords

hashcat -m 0 -a 0 -r best64.rule hashes.txt wordlist.txt - Rule application

john --rules=Jumbo hashes.txt - John rules engine

hashcat --stdout -a 0 -r rules.txt wordlist.txt - Generate candidates

Chapter 3: Rainbow Table Attacks

Rainbow tables provide time-memory trade-offs for password cracking

Rainbow tables are precomputed tables for reversing cryptographic hash functions, typically used for cracking password hashes by trading computational time for storage space.

Rainbow Table Tools and Usage

RainbowCrack

rcrack rainbow_tables/ -h hash_to_crack - Single hash lookup

rcrack rainbow_tables/ -l hash_list.txt - Multiple hash lookup

rtgen md5 lowercase 1 8 0 3800 33554432 0 - Generate tables

Ophcrack

GUI-based tool specializing in Windows password recovery using rainbow tables

Online Rainbow Tables

  • ‚Ä¢ HashKiller hash database
  • ‚Ä¢ MD5Decrypt online database
  • ‚Ä¢ Hashes.org distributed cracking

Chapter 5: Password Policy Assessment

Effective password policies balance security with usability

Password policy assessment involves evaluating organizational password requirements, implementation effectiveness, and user compliance to identify weaknesses and improvement opportunities.

Common Policy Weaknesses
  • ‚Ä¢ Minimum Length: Too short requirements (less than 12 characters)
  • ‚Ä¢ Complexity Rules: Predictable patterns and substitutions
  • ‚Ä¢ Expiration Policies: Frequent changes leading to weak passwords
  • ‚Ä¢ Reuse Restrictions: Insufficient password history depth
  • ‚Ä¢ Account Lockout: Inadequate protection against brute force
  • ‚Ä¢ Recovery Processes: Weak password reset mechanisms
Strong Policy Elements
  • ‚Ä¢ Length Requirements: Minimum 12-15 characters
  • ‚Ä¢ Passphrases: Encourage memorable long passwords
  • ‚Ä¢ Blacklists: Common password and breach databases
  • ‚Ä¢ Multi-Factor Authentication: Reduce password reliance
  • ‚Ä¢ Risk-Based: Adaptive requirements based on context
  • ‚Ä¢ User Education: Security awareness training
