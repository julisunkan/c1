Module 9: Wireless Network Security Testing

Wireless networks present unique security challenges and attack vectors. This module covers wireless security assessment techniques, from basic WEP cracking to advanced WPA3 attacks and rogue access point detection.

Learning Objectives

  • • Master wireless security protocols and their vulnerabilities
  • • Learn wireless reconnaissance and network discovery techniques
  • • Practice WEP, WPA, and WPA2/3 cracking methodologies
  • • Understand rogue access point detection and prevention
  • • Implement wireless monitoring and traffic analysis
  • • Execute advanced wireless attack scenarios

Wireless Security Protocols

Understanding wireless security protocols helps identify weaknesses and select appropriate attack methods.

Legacy Protocols

  • • WEP: Fundamentally broken, easily cracked
  • • WPS: PIN-based authentication vulnerabilities
  • • Open Networks: No encryption protection
  • • MAC Filtering: Easily bypassed access control

Modern Protocols

  • • WPA/WPA2: TKIP and AES encryption
  • • WPA3: Enhanced security with SAE
  • • Enterprise (802.1X): Certificate-based authentication
  • • OWE: Opportunistic Wireless Encryption

WEP Attacks

WEP (Wired Equivalent Privacy) has fundamental cryptographic flaws that make it trivial to crack.

Attack Methods

aircrack-ng capture.cap Statistical attack
aireplay-ng -1 0 -a [BSSID] wlan0mon Fake authentication
aireplay-ng -3 -b [BSSID] wlan0mon ARP replay attack

Attack Scenarios

  • • Passive Collection: Capture enough IVs
  • • Active Injection: Generate traffic for IVs
  • • Client-based: Target connected clients
  • • Fragmentation: Extract keystream

WPA/WPA2 Attacks

WPA/WPA2 attacks typically focus on capturing the 4-way handshake and cracking the Pre-Shared Key (PSK).

Handshake Capture and Cracking

Capture Handshake
airodump-ng -c [channel] --bssid [BSSID] -w capture wlan0mon
Force Deauthentication
aireplay-ng --deauth 10 -a [BSSID] wlan0mon
Crack PSK
aircrack-ng -w wordlist.txt capture.cap
Hashcat Cracking
hashcat -m 2500 capture.hccapx wordlist.txt

WPS Attacks

WiFi Protected Setup (WPS) vulnerabilities allow PIN-based attacks against WPA/WPA2 networks.

PIN Attack Tools

reaver -i wlan0mon -b [BSSID] -vv Reaver WPS attack
bully -b [BSSID] -c [channel] wlan0mon Bully WPS attack
wifite --wps Automated WPS testing

Attack Vectors

  • • PIN Brute Force: 11,000 possible combinations
  • • Pixie Dust: Weak random number generation
  • • Rate Limiting: Access point lockout mechanisms
  • • NULL PIN: Empty PIN acceptance

Enterprise Wireless Attacks

Enterprise wireless networks using 802.1X authentication present different attack vectors and challenges.

EAP Attacks

  • • EAP-MD5: Dictionary attacks on hashes
  • • PEAP-MSCHAPv2: Capture and crack challenges
  • • EAP-TLS: Certificate-based authentication
  • • EAP-FAST: Protected access credentials

Certificate Attacks

  • • Certificate Spoofing: Rogue RADIUS servers
  • • CA Validation: Trust anchor bypass
  • • Certificate Pinning: Client-side validation
  • • Credential Harvesting: Fake authentication

Wireless Defense Techniques

Understanding defensive measures helps assess the effectiveness of wireless security implementations.

Detection Systems

  • • WIDS/WIPS: Intrusion detection/prevention
  • • Rogue AP Detection: Unauthorized access points
  • • Spectrum Analysis: RF interference detection
  • • Client Monitoring: Device behavior analysis

Mitigation Strategies

  • • Strong Encryption: WPA3 implementation
  • • Certificate-based Auth: 802.1X deployment
  • • Network Segmentation: VLAN isolation
  • • Regular Updates: Firmware patching

Key Takeaways

Wireless network security testing requires understanding various protocols, attack vectors, and defense mechanisms. This module covered WEP/WPA/WPA2 attacks, WPS vulnerabilities, evil twin attacks, and enterprise wireless security. Modern wireless networks have significantly improved security, but legacy systems and misconfigurations still present opportunities for attackers. Always test wireless security with proper authorization and consider the impact on legitimate users. Regular security assessments help organizations maintain strong wireless security postures and detect potential threats.
