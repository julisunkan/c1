Information Gathering and Reconnaissance

Learn passive and active information gathering techniques to map target environments

Introduction

Information gathering, also known as reconnaissance, is the first and most critical phase of penetration testing. This phase involves collecting as much information as possible about the target organization, systems, and infrastructure before launching any active attacks. Proper reconnaissance can make the difference between a successful penetration test and a failed attempt.

Learning Objectives

  • Understand the difference between passive and active reconnaissance
  • Learn OSINT (Open Source Intelligence) techniques
  • Master DNS enumeration and subdomain discovery
  • Perform social media and employee reconnaissance
  • Gather technical infrastructure information
  • Document findings systematically
  • Stay within legal and ethical boundaries

DNS Enumeration

DNS provides valuable information about target infrastructure:

Basic DNS Queries

# DNS record types nslookup example.com dig example.com dig example.com MX dig example.com NS dig example.com TXT dig example.com ANY # Reverse DNS lookup dig -x 192.168.1.1 nslookup 192.168.1.1

DNS Zone Transfer

# Attempt zone transfer dig axfr example.com @ns1.example.com dig axfr @ns1.example.com example.com # Find name servers first dig example.com NS

Subdomain Enumeration

Sublist3r - Python tool for subdomain enumeration
sublist3r -d example.com sublist3r -d example.com -b -t 100
Amass - Advanced subdomain enumeration
amass enum -d example.com amass enum -d example.com -brute
Gobuster - Directory and subdomain brute forcer
gobuster dns -d example.com -w /usr/share/wordlists/SecLists/Discovery/DNS/subdomains-top1million-5000.txt

WHOIS Information

# WHOIS queries whois example.com whois 192.168.1.1 # Extract useful information: # - Registrar details # - Registration dates # - Contact information # - Name servers # - IP ranges

Reconnaissance Framework

  1. Define Scope: Clearly identify what's in scope
  2. Passive Information Gathering: Start with OSINT
  3. Active Information Gathering: Direct target interaction
  4. Analysis and Correlation: Connect the dots
  5. Documentation: Record all findings
  6. Vulnerability Identification: Identify potential attack vectors

Documentation and Note-Taking

# Create organized directory structure mkdir -p recon/{osint,dns,subdomains,emails,social,technical} # Document findings systematically echo "Domain: example.com" > recon/target_info.txt echo "Discovered subdomains:" >> recon/subdomains/subdomains.txt echo "Employee emails:" >> recon/emails/employees.txt

Automation and Scripting

#!/bin/bash # Basic reconnaissance script target=$1 echo "Starting reconnaissance for $target" # Subdomain enumeration echo "Finding subdomains..." sublist3r -d $target -o subdomains.txt # DNS enumeration echo "Performing DNS enumeration..." dig $target ANY > dns_records.txt # WHOIS lookup echo "Getting WHOIS information..." whois $target > whois_info.txt echo "Reconnaissance complete!"
Legal and Ethical Considerations:
  • Always obtain proper authorization before testing
  • Respect rate limits and terms of service
  • Don't access or download sensitive information
  • Use VPNs and proxies appropriately
  • Document your methodology for transparency
