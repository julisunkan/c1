Module 29: Capstone Project - Comprehensive Penetration Test

Learning Objectives

  • Apply comprehensive penetration testing methodology to a complex scenario
  • Demonstrate mastery of technical skills across multiple domains
  • Develop professional-grade penetration testing reports
  • Practice real-world client communication and project management
  • Integrate ethical and legal considerations into testing activities

Project Overview

This capstone project simulates a comprehensive penetration testing engagement for "TechCorp Industries," a fictional technology company. You'll conduct a full-scope assessment including external and internal network testing, web application analysis, and social engineering evaluation.

Project Scenario

Client Background: TechCorp Industries is a software development company with 500 employees across three locations. They develop custom enterprise software and host client applications in their data center and cloud environments.

Engagement Scope

  • External Network: Internet-facing infrastructure and services
  • Internal Network: Corporate LAN and server environments
  • Web Applications: Customer portal and internal applications
  • Wireless Networks: Corporate and guest Wi-Fi infrastructure
  • Social Engineering: Limited phishing and physical security testing

Project Phases

Phase 1: Planning and Reconnaissance (Week 1)

Deliverables:

  • Detailed test plan and methodology
  • Risk assessment and safety procedures
  • Passive information gathering report
  • Attack surface analysis
# Reconnaissance Activities
# OSINT gathering
theHarvester -d techcorp.com -b all
recon-ng
shodan search org:"TechCorp Industries"

# DNS enumeration
dnsrecon -d techcorp.com -t axfr
fierce --domain techcorp.com

# Social media and document analysis
google-dorking "site:techcorp.com filetype:pdf"
exiftool *.pdf

Phase 2: External Assessment (Week 2)

Testing Activities:

  • Network scanning and service enumeration
  • Vulnerability identification and validation
  • Web application security testing
  • Email and DNS security assessment
# External network assessment
nmap -sS -sV -sC -O -A --script vuln 203.0.113.0/24
masscan -p1-65535 203.0.113.0/24 --rate=1000

# Web application testing
nikto -h https://portal.techcorp.com
dirb https://portal.techcorp.com /usr/share/dirb/wordlists/common.txt
sqlmap -u "https://portal.techcorp.com/login" --forms --dbs

# SSL/TLS assessment
sslyze --regular portal.techcorp.com
testssl.sh portal.techcorp.com

Key Findings Template:

Critical Finding Example:
  • Vulnerability: SQL Injection in login form
  • Risk Level: Critical
  • Impact: Complete database compromise
  • Proof of Concept: [Screenshot and steps]
  • Recommendation: Implement parameterized queries

Phase 3: Internal Assessment (Week 3)

Testing Activities:

  • Internal network enumeration
  • Active Directory security assessment
  • Privilege escalation and lateral movement
  • Data exfiltration simulation
# Internal network reconnaissance
nmap -sn 10.0.0.0/8

# Active Directory enumeration
bloodhound-python -d techcorp.local -u user -p password -ns 192.168.1.5

# Privilege escalation
winPEAS.exe
linpeas.sh
sudo -l

Advanced Assessment Areas:

Reporting Requirements

Executive Summary Structure

  • Assessment Overview: Scope, methodology, and timeline
  • Key Findings: High-level security posture assessment
  • Risk Summary: Categorized vulnerability statistics
  • Business Impact: Potential consequences of identified risks
  • Recommendations: Prioritized remediation roadmap

Technical Report Components

Required Sections:
  • Methodology and approach
  • Detailed findings with proof of concept
  • Risk ratings and CVSS scores
  • Remediation recommendations
  • Appendices with tool outputs

Risk Rating Matrix

RISK CALCULATION FORMULA:
Risk = Likelihood Ã— Impact

LIKELIHOOD SCALE:
- Low (1): Requires specialized skills and access
- Medium (2): Requires some technical knowledge
- High (3): Can be exploited by script kiddies

IMPACT SCALE:
- Low (1): Minimal business impact
- Medium (2): Moderate business disruption
- High (3): Severe business consequences

RISK LEVELS:
- Critical (8-9): Immediate action required
- High (6-7): Remediate within 30 days
- Medium (4-5): Remediate within 90 days
- Low (1-3): Remediate when possible
