Static Analysis Techniques

Android Static Analysis

# Android static analysis workflow # 1. APK information gathering aapt dump badging app.apk aapt dump permissions app.apk aapt dump configurations app.apk # 2. APK extraction and decompilation # Extract APK contents unzip app.apk -d extracted_apk/ # Decompile with APKTool apktool d app.apk -o decompiled_app # Decompile to Java with JADX jadx -d output_dir app.apk # 3. Manifest analysis # Check AndroidManifest.xml for: # - Permissions # - Exported components # - Intent filters # - Debugging flags grep -r "android:exported" decompiled_app/ grep -r "android:allowBackup" decompiled_app/ grep -r "android:debuggable" decompiled_app/ # 4. Source code analysis # Search for sensitive information grep -r "password\|api_key\|secret" decompiled_app/ grep -r "http://\|https://" decompiled_app/ grep -r "crypto\|encrypt\|decrypt" decompiled_app/ # Find hardcoded credentials grep -r "SharedPreferences\|getSharedPreferences" decompiled_app/ grep -r "SQLiteDatabase\|openOrCreateDatabase" decompiled_app/ # 5. Native library analysis # Extract and analyze .so files find extracted_apk/ -name "*.so" strings lib/arm64-v8a/native_lib.so objdump -d lib/arm64-v8a/native_lib.so # 6. Certificate analysis # Extract certificate from APK keytool -printcert -jarfile app.apk # 7. Automated static analysis # MobSF static analysis curl -F "file=@app.apk" http://localhost:8000/api/v1/upload # QARK (Quick Android Review Kit) python3 qark.py --apk app.apk # Semgrep for mobile security semgrep --config=auto decompiled_app/

Android Dynamic Analysis

# Android dynamic analysis setup and execution # 1. Device preparation # Enable developer options and USB debugging adb devices adb shell # Root device (if needed) # Install Magisk or SuperSU # 2. Frida setup # Install frida-server on device adb push frida-server /data/local/tmp/ adb shell chmod 755 /data/local/tmp/frida-server adb shell /data/local/tmp/frida-server & # 3. Runtime analysis with Frida # List running processes frida-ps -U # Attach to application frida -U -n com.example.app # JavaScript hooking example frida -U -l hook_script.js com.example.app # Sample Frida script for Android cat > android_hooks.js << 'EOF' Java.perform(function() { // Hook Java methods var ActivityClass = Java.use("android.app.Activity"); ActivityClass.onCreate.implementation = function(savedInstanceState) { console.log("[+] Activity.onCreate() called"); this.onCreate(savedInstanceState); }; // Hook crypto operations var CipherClass = Java.use("javax.crypto.Cipher"); CipherClass.doFinal.overload('[B').implementation = function(input) { console.log("[+] Cipher.doFinal() called with: " + input); return this.doFinal(input); }; }); EOF # 4. Network traffic analysis # Set up proxy (Burp Suite/OWASP ZAP) adb shell settings put global http_proxy proxy_ip:8080 # SSL pinning bypass with Frida frida -U --codeshare akabe1/frida-multiple-unpinning -l ssl_bypass.js com.example.app # 5. File system monitoring # Monitor file operations adb shell strace -e trace=file -p $(pidof com.example.app) # 6. Memory analysis # Dump application memory adb shell su -c "cat /proc/$(pidof com.example.app)/maps" adb shell su -c "dd if=/proc/$(pidof com.example.app)/mem of=/sdcard/mem_dump.bin" # 7. Intent fuzzing # Send malicious intents adb shell am start -n com.example.app/.MainActivity --es "key" "malicious_value" adb shell am broadcast -a com.example.ACTION --es "data" "../../../etc/passwd" # 8. Database analysis # Extract and analyze SQLite databases adb shell su -c "cp /data/data/com.example.app/databases/app.db /sdcard/" adb pull /sdcard/app.db sqlite3 app.db .dump

Mobile Security Best Practices

Secure Development Guidelines

üîí Data Protection

  • Encrypt sensitive data at rest
  • Use secure key management
  • Minimize data collection
  • Implement data retention policies
  • Secure data transmission

üîë Authentication & Authorization

  • Implement strong authentication
  • Use biometric authentication when available
  • Implement proper session management
  • Use OAuth 2.0/OpenID Connect
  • Implement proper access controls

üåê Network Security

  • Use TLS for all communications
  • Implement certificate pinning
  • Validate SSL/TLS certificates
  • Avoid mixed content issues
  • Implement proper timeout handling

üõ°Ô∏è Runtime Protection

  • Implement anti-tampering measures
  • Use code obfuscation
  • Implement root/jailbreak detection
  • Use runtime application self-protection (RASP)
  • Implement integrity checks
Legal and Ethical Considerations:
  • Only test applications you own or have authorization to test
  • Respect user privacy and data protection regulations
  • Use test devices isolated from production networks
  • Follow responsible disclosure practices for vulnerabilities
  • Be aware of app store policies and legal requirements
  • Protect sensitive data discovered during testing
  • Document all testing activities and findings
Key Takeaway: Mobile application security requires understanding platform-specific security models, comprehensive testing methodologies, and awareness of mobile-specific attack vectors. Effective mobile security combines secure development practices, thorough testing, and ongoing monitoring to protect against evolving mobile threats.
