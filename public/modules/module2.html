File System Structure

Understanding the Linux file system hierarchy is essential for locating important files during security assessments.

System Directories

  • • /etc: System configuration files
  • • /var: Variable data (logs, databases)
  • • /usr: User programs and libraries
  • • /home: User home directories
  • • /root: Root user home directory
  • • /tmp: Temporary files

Important Files

  • • /etc/passwd: User account information
  • • /etc/shadow: Password hashes
  • • /etc/group: Group information
  • • /var/log/: System log files
  • • /etc/crontab: Scheduled tasks
  • • /etc/hosts: Local hostname resolution

Shell Scripting Basics

Basic shell scripting skills help automate repetitive tasks and create custom tools for penetration testing.

Basic Script Structure

#!/bin/bash

# Variable assignment
TARGET_IP="192.168.1.1"
PORT_RANGE="1-1000"

# Function definition
scan_ports() {
    echo "Scanning $1 on ports $2"
    nmap -p $2 $1
}

# Conditional statement
if [ $# -eq 0 ]; then
    echo "Usage: $0 "
    exit 1
fi

# Loop example
for port in 21 22 23 25 53 80 443; do
    echo "Checking port $port"
    nc -zv $1 $port 2>&1 | grep succeeded
done

# Call function
scan_ports $TARGET_IP $PORT_RANGE

Security Hardening Basics

Understanding security hardening helps you identify weaknesses and recommend improvements.

User Account Security

  • • Disable unnecessary user accounts
  • • Enforce strong password policies
  • • Configure account lockout policies
  • • Use sudo instead of root login
  • • Monitor user activity logs

System Security

  • • Keep system and packages updated
  • • Configure firewall rules (iptables/ufw)
  • • Disable unnecessary services
  • • Secure SSH configuration
  • • Enable audit logging

Common Security Misconfigurations

Recognizing common misconfigurations helps you identify potential attack vectors during assessments.

Frequent Issues to Check

  • • World-writable files: find / -perm -002 -type f 2>/dev/null
  • • SUID/SGID files: find / -perm -4000 -o -perm -2000 2>/dev/null
  • • Default passwords: Check for unchanged default credentials
  • • Open network services: Unnecessary services listening on network
  • • Weak file permissions: Sensitive files readable by all users
  • • Outdated software: Unpatched systems and applications
