Network Fundamentals

OSI Model and Security

The OSI (Open Systems Interconnection) model provides a framework for understanding network communications and security at each layer:

Layer Name Security Concerns Protections
7 Application Malware, Data breaches Application firewalls, Input validation
6 Presentation Encryption weaknesses Strong encryption, SSL/TLS
5 Session Session hijacking Session management, Authentication
4 Transport Port scanning, DoS Firewalls, Rate limiting
3 Network IP spoofing, Routing attacks Routers, Network ACLs
2 Data Link MAC flooding, VLAN hopping Switch security, VLANs
1 Physical Cable tapping, Physical access Physical security, Cable protection

TCP/IP Protocol Suite

Understanding the security implications of core protocols:

Internet Protocol (IP)

  • IPv4: 32-bit addressing, limited security features
  • IPv6: 128-bit addressing, built-in IPSec support
  • Vulnerabilities: IP spoofing, fragmentation attacks

Transmission Control Protocol (TCP)

  • Connection-oriented, reliable delivery
  • Three-way handshake process
  • Vulnerabilities: SYN flooding, sequence number attacks

User Datagram Protocol (UDP)

  • Connectionless, faster but unreliable
  • No built-in error correction
  • Vulnerabilities: UDP flooding, amplification attacks
Security Principle: Defense in depth requires implementing security controls at multiple layers of the network stack.

Network Security Devices

Firewalls

Network security devices that monitor and control incoming and outgoing network traffic based on predetermined security rules.

Types of Firewalls

  • Packet Filtering: Examines packets and allows/blocks based on rules
  • Stateful Inspection: Tracks connection states and context
  • Application Layer: Deep packet inspection at application layer
  • Next-Generation: Combines multiple security functions

Firewall Rules

Example firewall rule structure:

Action: ALLOW/DENY
Source: 192.168.1.0/24
Destination: 10.0.0.5
Protocol: TCP
Port: 80

Intrusion Detection Systems (IDS)

Monitor network traffic for suspicious activity and known threats.

Types of IDS

  • Network-based (NIDS): Monitor network traffic
  • Host-based (HIDS): Monitor individual systems
  • Signature-based: Detect known attack patterns
  • Anomaly-based: Detect deviations from normal behavior

Intrusion Prevention Systems (IPS)

Active security devices that can automatically block detected threats.

  • Real-time threat blocking
  • Integration with firewalls
  • Automated response capabilities

Network Vulnerabilities and Attacks

Common Network Attacks

Denial of Service (DoS) Attacks

  • SYN Flood: Overwhelm target with connection requests
  • Ping Flood: Overwhelm with ICMP packets
  • UDP Flood: Flood target with UDP packets
  • DDoS: Distributed attack from multiple sources

Man-in-the-Middle (MITM) Attacks

  • ARP spoofing
  • DNS spoofing
  • SSL stripping
  • Rogue access points

Network Reconnaissance

  • Port Scanning: Discovering open ports and services
  • Network Mapping: Identifying network topology
  • OS Fingerprinting: Determining operating systems
  • Service Enumeration: Identifying running services
Legal Warning: Port scanning and network reconnaissance activities should only be performed on networks you own or have explicit permission to test.

Network Segmentation

Virtual LANs (VLANs)

Logical segmentation of networks to improve security and performance:

  • Isolate sensitive systems
  • Reduce broadcast domains
  • Implement access controls
  • Contain security breaches

Demilitarized Zone (DMZ)

A perimeter network that sits between internal networks and external networks:

  • Hosts public-facing services
  • Provides buffer zone
  • Reduces internal network exposure
  • Enables controlled access

Network Access Control (NAC)

Security approach that enforces policy on devices seeking network access:

  • Device authentication
  • Compliance checking
  • Policy enforcement
  • Quarantine capabilities

Secure Network Protocols

SSL/TLS

Cryptographic protocols that provide secure communication over networks:

  • SSL 3.0: Deprecated due to vulnerabilities
  • TLS 1.2: Widely supported, secure
  • TLS 1.3: Latest version, improved security and performance

IPSec

Protocol suite for securing Internet Protocol communications:

  • Authentication Header (AH): Provides authentication
  • Encapsulating Security Payload (ESP): Provides encryption
  • Tunnel Mode: Protects entire IP packet
  • Transport Mode: Protects payload only

SSH (Secure Shell)

Protocol for secure remote access and command execution:

  • Encrypted communication
  • Strong authentication
  • Port forwarding capabilities
  • File transfer support (SCP/SFTP)
Best Practice: Always use the latest versions of secure protocols and disable legacy protocols with known vulnerabilities.

Wireless Network Security

Wi-Fi Security Protocols

  • WEP: Deprecated, easily broken
  • WPA: Improved security over WEP
  • WPA2: Current standard, uses AES encryption
  • WPA3: Latest standard, enhanced security features

Wireless Threats

  • Rogue access points
  • Evil twin attacks
  • Packet sniffing
  • Deauthentication attacks

Network Monitoring and Analysis

Network Monitoring Tools

  • Wireshark: Network protocol analyzer
  • tcpdump: Command-line packet analyzer
  • Nmap: Network discovery and security auditing
  • Netstat: Display network connections and statistics

Log Analysis

Key logs to monitor for security events:

  • Firewall logs
  • IDS/IPS alerts
  • Router and switch logs
  • DHCP server logs
  • DNS server logs

Network Security Best Practices

  1. Implement Defense in Depth: Multiple layers of security controls
  2. Regular Security Assessments: Vulnerability scans and penetration testing
  3. Keep Systems Updated: Apply security patches promptly
  4. Use Strong Authentication: Multi-factor authentication where possible
  5. Monitor Network Traffic: Continuous monitoring for anomalies
  6. Segment Networks: Isolate critical systems and data
  7. Encrypt Sensitive Data: Both in transit and at rest
  8. Incident Response Plan: Prepare for security incidents

Conclusion

Network security is fundamental to protecting organizational assets and maintaining business continuity. Understanding network protocols, security devices, and common threats enables you to design and implement effective network security architectures.

Key Takeaway: Network security requires a comprehensive approach combining technology, processes, and people to effectively protect against evolving threats.
