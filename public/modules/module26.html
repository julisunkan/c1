Cryptographic Fundamentals

Cryptography assessment involves evaluating the implementation and configuration of cryptographic systems, protocols, and Public Key Infrastructure. This includes analyzing encryption algorithms, key management practices, and certificate handling.

Common Cryptographic Vulnerabilities

  • Weak Encryption Algorithms: Use of deprecated or weak ciphers (DES, RC4)
  • Poor Key Management: Hardcoded keys, weak key generation
  • Implementation Flaws: Side-channel attacks, timing attacks
  • Protocol Weaknesses: SSL/TLS misconfigurations
  • Certificate Issues: Self-signed certificates, weak validation

PKI Assessment Methodology

Certificate Analysis

# Certificate examination with OpenSSL
openssl x509 -in certificate.crt -text -noout
openssl x509 -in certificate.crt -purpose -noout

# Check certificate chain
openssl verify -CAfile ca-bundle.crt certificate.crt

# Test SSL/TLS configuration
openssl s_client -connect target.com:443 -cipher 'ALL:!aNULL'

SSL/TLS Security Testing

  • SSLyze: Comprehensive SSL/TLS scanner
  • testssl.sh: Command-line SSL/TLS tester
  • sslscan: Fast SSL/TLS security scanner
  • nmap SSL scripts: Built-in SSL/TLS testing capabilities
# SSL/TLS testing with various tools
sslyze --regular target.com:443
testssl.sh target.com
sslscan target.com:443
nmap --script ssl-enum-ciphers -p 443 target.com

Cryptographic Protocol Testing

Common Protocol Vulnerabilities

  • BEAST Attack: Browser Exploit Against SSL/TLS
  • CRIME/BREACH: Compression-based attacks
  • POODLE: Padding Oracle On Downgraded Legacy Encryption
  • Heartbleed: OpenSSL buffer over-read vulnerability
  • FREAK: Factoring RSA Export Keys attack

Advanced Cryptographic Analysis

# Check for specific vulnerabilities
nmap --script ssl-heartbleed target.com
nmap --script ssl-poodle target.com
nmap --script ssl-ccs-injection target.com

# Analyze cipher suites
nmap --script ssl-enum-ciphers -p 443 target.com | grep -E "(weak|broken|insecure)"

Key Management Assessment

Key Storage Security

  • Hardware Security Modules (HSM) evaluation
  • Key escrow and recovery procedures
  • Key rotation and lifecycle management
  • Access controls and authentication

Cryptographic Implementation Review

Critical: Never rely solely on encryption algorithms. Always verify proper implementation, key management, and protocol configuration for comprehensive security.

Practical Assessment Tools

Specialized Cryptographic Tools

  • HashCat: Advanced password recovery and hash cracking
  • John the Ripper: Password cracking with cryptographic support
  • CrypTool: Educational cryptography and cryptanalysis
  • SAGE: Mathematical software for cryptographic analysis
# Hash analysis and cracking
hashcat -m 1000 -a 0 hashes.txt wordlist.txt
john --format=NT hashes.txt

# Certificate and key analysis
openssl rsa -in private.key -text -noout
openssl req -in csr.csr -text -noout

PKI Infrastructure Testing

Certificate Authority (CA) Assessment

  • Root CA security evaluation
  • Certificate issuance validation
  • Revocation mechanisms (CRL, OCSP)
  • Cross-certification analysis

Certificate Validation Testing

# OCSP testing
openssl ocsp -issuer ca.crt -cert certificate.crt -url http://ocsp.example.com

# CRL checking
openssl crl -in crl.pem -text -noout

# Certificate transparency logs
curl -s "https://ct.googleapis.com/logs/argon2023/ct/v1/get-entries?start=0&end=0"
Best Practice: Regularly audit cryptographic implementations and configurations. Cryptographic security requires ongoing assessment as new vulnerabilities and attack techniques emerge.
Remember: Cryptographic assessment requires both technical analysis and understanding of business requirements. Balance security with usability and performance considerations.
